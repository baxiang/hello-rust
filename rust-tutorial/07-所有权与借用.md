# 第 07 章：所有权系统

> Rust 最独特的特性，内存安全的核心保障——无需垃圾回收器

## 7.1 为什么需要所有权？

### 其他语言的内存管理方式

```
┌─────────────────────────────────────────────────────┐
│              内存管理方式对比                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  C/C++：手动管理                                     │
│  ├── 程序员负责分配/释放                            │
│  ├── 灵活但容易出错                                 │
│  └── 常见问题：内存泄漏、悬垂指针、双重释放          │
│                                                     │
│  Java/Python/Go：垃圾回收（GC）                      │
│  ├── 运行时自动回收                                 │
│  ├── 安全但有性能开销                               │
│  └── 问题：GC 暂停、不可预测的延迟                   │
│                                                     │
│  Rust：所有权系统                                    │
│  ├── 编译时检查                                     │
│  ├── 零运行时开销                                   │
│  └── 安全且高效！                                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 所有权解决的问题

```rust
// C 风格：内存泄漏
void create_leak() {
    char* buffer = malloc(100);
    // 忘记 free，内存泄漏！
}

// Rust：编译时自动释放
fn no_leak() {
    let buffer = String::with_capacity(100);
    // 离开作用域时自动释放！
}
```

## 7.2 所有权的三大规则

```
┌─────────────────────────────────────────────────────┐
│              所有权三定律                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  第一定律：每个值都有一个所有者                     │
│           Each value has an owner                  │
│                                                     │
│  第二定律：任一时刻只能有一个所有者                │
│           There can only be one owner at a time    │
│                                                     │
│  第三定律：当所有者离开作用域，值会被丢弃          │
│           When the owner goes out of scope,        │
│           the value will be dropped                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 规则详解

```rust
fn main() {
    // 第一定律：每个值都有一个所有者
    let s = String::from("hello");  // s 是 "hello" 的所有者

    {
        // 第二定律：任一时刻只能有一个所有者
        let t = s;  // 所有权从 s 移动到 t
        // 现在 t 是所有者，s 不再有效

        println!("{}", t);  // ✅ 正确
        // println!("{}", s);  // ❌ 错误：s 已无效
    }
    // t 离开作用域

    // 第三定律：所有者离开作用域，值被丢弃
    let u = String::from("world");
}  // u 离开作用域，"world" 被自动释放
```

## 7.3 作用域（Scope）详解

### 作用域基础

```rust
fn main() {
    // s 从这里开始有效
    let s = String::from("hello");

    println!("{}", s);  // 可以使用 s
}  // s 离开作用域，内存被自动释放
   // Rust 在这里隐式调用 drop(s)
```

### 作用域嵌套

```rust
fn main() {
    let outer = String::from("外层");

    {
        let inner = String::from("内层");
        println!("inner: {}", inner);  // ✅ 可以
        println!("outer: {}", outer);  // ✅ 也可以
    }  // inner 离开作用域，被释放

    println!("outer: {}", outer);  // ✅ outer 仍然有效
    // println!("inner: {}", inner);  // ❌ inner 已无效
}  // outer 离开作用域，被释放
```

### 作用域与内存释放时机

```rust
fn main() {
    let s1 = String::from("first");
    let s2 = String::from("second");
    let s3 = String::from("third");

    // s1, s2, s3 都在作用域中

    {
        let s4 = String::from("fourth");
        println!("s4: {}", s4);
    }  // s4 在这里被释放

    println!("s1: {}", s1);
    println!("s2: {}", s2);
    println!("s3: {}", s3);
}  // s3, s2, s1 按相反顺序被释放
   // 释放顺序：s3 → s2 → s1（后进先出）
```

## 7.4 String 类型详解

### String vs &str

这是 Rust 初学者最容易困惑的地方，我们详细对比：

```
┌─────────────────────────────────────────────────────┐
│           String vs &str 详细对比                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  String（拥有所有权的字符串）                        │
│  ├── 可增长、可修改                                  │
│  ├── 数据存储在堆上                                 │
│  ├── 大小：24 字节（指针 + 长度 + 容量）              │
│  ├── 创建：String::from("hello")                    │
│  └── 所有权转移时会影响性能（移动）                  │
│                                                     │
│  &str（字符串切片/视图）                            │
│  ├── 不可变（通常）                                  │
│  ├── 借用数据，不拥有所有权                         │
│  ├── 大小：16 字节（指针 + 长度）                     │
│  ├── 字面量："hello"（类型是 &'static str）          │
│  └── 传递时只复制指针，非常高效                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 内存布局可视化

```
字符串字面量 &str：
┌─────────────────────────────────────────┐
│           栈（Stack）                    │
│  ┌───────────────┬───────────────┐      │
│  │    指针       │     长度      │      │
│  │  (指向静态区) │    (5)        │      │
│  └───────┬───────┴───────────────┘      │
│          │                               │
│          ▼                               │
│  ┌─────────────────────────────────┐    │
│  │       静态区（编译时常量）        │    │
│  │  "hello\0"                      │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘

String（堆分配字符串）：
┌─────────────────────────────────────────┐
│           栈（Stack）                    │
│  ┌───────────┬───────────┬─────────┐   │
│  │   指针    │   长度    │  容量   │   │
│  │ (指向堆)  │   (5)     │  (10)   │   │
│  └─────┬─────┴───────────┴─────────┘   │
│        │                                │
│        ▼                                │
│  ┌─────────────────────────────────┐   │
│  │       堆（Heap）                 │   │
│  │  "hello"  │  空闲空间 (5)       │   │
│  └─────────────────────────────────┘   │
│                                         │
│  可以增长：push_str(" world")           │
└─────────────────────────────────────────┘
```

### 代码示例

```rust
fn main() {
    // &str - 字符串字面量
    let s1: &str = "hello";

    // String - 可增长字符串
    let s2: String = String::from("hello");
    let s3: String = "hello".to_string();
    let s4: String = "hello".to_owned();

    // 修改 String
    let mut s5 = String::from("hello");
    s5.push_str(" world!");
    println!("{}", s5);  // hello world!

    // &str 不能修改
    // s1.push_str(" world");  // ❌ 错误

    // 类型验证
    println!("s1 类型：{:?}", std::any::type_name_of_val(&s1));
    println!("s2 类型：{:?}", std::any::type_name_of_val(&s2));
}
```

## 7.5 移动语义（Move）

### 移动 vs 复制

```rust
fn main() {
    // 对于 String 类型：移动语义
    let s1 = String::from("hello");
    let s2 = s1;  // 移动所有权

    // println!("{}", s1);  // ❌ 错误：s1 已无效
    println!("{}", s2);  // ✅ 正确

    // 对于整数类型：复制语义（Copy trait）
    let x = 5;
    let y = x;  // 复制值

    println!("x = {}, y = {}", x, y);  // 都有效
}
```

### 移动语义图解

```
移动前：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
└──────┘            └─────────┘

let s2 = s1;  之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ (无效)     │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            │         │
│ s2   │ ───────────┘         │
│ ptr  │                      │
│ len  │                      │
│ cap  │                      │
└──────┘                      │

关键点：                       │
• s2 获得了指向堆数据的指针     │
• s1 被标记为无效（不能使用）   │
• 数据没有被复制！              │
• 当 s2 离开作用域时，数据被释放 │
```

### 为什么 Rust 不自动深拷贝？

```
假设 Rust 自动深拷贝：
let s1 = String::from("hello");
let s2 = s1;  // 如果自动复制堆数据...

问题：
┌─────────────────────────────────────────┐
│ 1. 性能问题                              │
│    - 堆数据可能很大（几 MB 甚至几 GB）     │
│    - 每次赋值都要复制，非常慢           │
│    - 程序员可能不知道发生了昂贵的复制   │
├─────────────────────────────────────────┤
│ 2. 所有权不明确                          │
│    - s1 和 s2 都指向相同的数据？          │
│    - 谁负责释放？两个都释放？双重释放！  │
│    - 只有一个释放？另一个怎么办？        │
└─────────────────────────────────────────┘

Rust 的选择：
• 默认移动（零成本）
• 需要复制时显式调用 clone()
• 明确、可控、高效
```

### 显式深拷贝：clone()

```rust
fn main() {
    let s1 = String::from("hello");

    // 显式深拷贝
    let s2 = s1.clone();

    // 现在 s1 和 s2 都有自己独立的堆数据
    println!("s1 = {}, s2 = {}", s1, s2);  // 都有效

    // 修改 s2 不影响 s1
    let mut s2 = s2;
    s2.push_str(" world");
    println!("s1 = {}", s1);  // hello
    println!("s2 = {}", s2);  // hello world
}
```

### clone() 的内存布局

```
clone() 之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            └─────────┘
│ s2   │            ┌─────────┐
│ ptr  │ ─────────→ │ "hello" │
│ len  │            │ (副本)  │
│ cap  │            │         │
└──────┘            └─────────┘

关键点：
• s1 和 s2 指向不同的堆数据
• 数据被完整复制了一份
• 两者互不影响
• 各自在离开作用域时释放自己的数据
```

## 7.6 函数与所有权

### 参数传递：移动所有权

```rust
fn main() {
    let s = String::from("hello");
    takes_ownership(s);  // s 的所有权被移动

    // println!("{}", s);  // ❌ 错误：s 已无效

    let x = 5;
    makes_copy(x);  // x 被复制（i32 实现 Copy）

    println!("x = {}", x);  // ✅ 正确：x 仍然有效
}

fn takes_ownership(some_string: String) {
    // some_string 获得所有权
    println!("{}", some_string);
}  // some_string 离开作用域，数据被释放

fn makes_copy(some_integer: i32) {
    // some_integer 是 x 的副本
    println!("{}", some_integer);
}  // 没有堆数据，无需释放
```

### 返回值：转移所有权

```rust
fn main() {
    // 函数返回所有权
    let s1 = gives_ownership();
    // s1 现在拥有 "hello" 的所有权

    let s2 = String::from("world");
    let s3 = takes_and_gives_back(s2);
    // s2 的所有权移入函数，然后 s3 获得返回的所有权
}

fn gives_ownership() -> String {
    let some_string = String::from("hello");
    some_string  // 返回，所有权转移给调用者
}

fn takes_and_gives_back(a_string: String) -> String {
    // a_string 获得所有权
    a_string  // 返回，所有权转移给调用者
}
```

### 所有权流动图解

```
fn gives_ownership() -> String {
    let s = String::from("hello");
    s  // ← 返回
}

fn main() {
    let s1 = gives_ownership();
    //     ↑
    //     所有权从函数流向 main
}
```

### 实际模式：多次使用变量

```rust
// ❌ 不好：每次都移动
fn main() {
    let s = String::from("hello");
    let s = take_and_return(s);  // 取回所有权
    let s = take_and_return(s);  // 再次取回
}

fn take_and_return(s: String) -> String {
    println!("{}", s);
    s  // 返回
}

// ✅ 更好：使用引用（下一章详细讲解）
fn main() {
    let s = String::from("hello");
    just_borrow(&s);  // 不移动
    just_borrow(&s);  // 可以再次使用
}

fn just_borrow(s: &String) {
    println!("{}", s);
}

// ✅ 最佳：接受 &str
fn main() {
    let s = String::from("hello");
    print_str(&s);  // &String → &str
    print_str("literal");  // &'static str
}

fn print_str(s: &str) {
    println!("{}", s);
}
```

## 7.7 Copy trait

### 什么是 Copy trait？

实现了 `Copy` trait 的类型，赋值时自动复制而不是移动：

```rust
fn main() {
    let x = 5;
    let y = x;  // 自动复制，不是移动

    println!("x = {}, y = {}", x, y);  // x 仍然有效
}
```

### Copy 类型列表

```rust
fn main() {
    // 所有整数类型
    let a: i8 = 1;
    let b: i16 = 2;
    let c: i32 = 3;
    let d: i64 = 4;
    let e: i128 = 5;
    let f: isize = 6;  // 指针大小的有符号整数
    let g: u8 = 7;
    let h: u16 = 8;
    let i: u32 = 9;
    let j: u64 = 10;
    let k: u128 = 11;
    let l: usize = 12;  // 指针大小的无符号整数

    // 所有浮点类型
    let m: f32 = 3.14;
    let n: f64 = 2.71;

    // 布尔类型
    let o: bool = true;

    // 字符类型
    let p: char = 'A';

    // 所有元素都 Copy 的元组
    let q: (i32, i32) = (1, 2);
    let r = q;
    println!("q = {:?}", q);  // ✅ q 仍然有效

    // 所有元素都 Copy 的数组
    let s: [i32; 3] = [1, 2, 3];
    let t = s;
    println!("s = {:?}", s);  // ✅ s 仍然有效
}
```

### 为什么元组和数组可以是 Copy 类型？

这是 Rust 初学者常问的问题。我们来详细解释其中的原理。

#### 核心判断标准

一个类型要实现 `Copy` trait，必须满足一个关键条件：

> **类型本身不包含堆上分配的数据，且其所有组成部分也都是 Copy 的**

#### 元组的 Copy 规则

元组（Tuple）是否 Copy **取决于其所有元素的类型**：

```rust
// ✅ 这些元组是 Copy 的，因为所有元素都是 Copy
let t1 = (1, 2, 3);              // (i32, i32, i32)
let t2 = (1, 'a', true);         // (i32, char, bool)
let t3 = (1.5, 2.5);             // (f64, f64)

let a = t1;
println!("{:?}", t1);  // ✅ t1 仍然有效，因为是位复制
```

```rust
// ❌ 这个元组 NOT Copy，因为包含非 Copy 元素
let t4 = (String::from("hello"), 5);  // (String, i32)

let b = t4;
// println!("{:?}", t4);  // ❌ 错误：t4 已移动
```

**内存布局图解：**

```
Copy 元组 (i32, i32)：
┌─────────────────────────┐
│         栈 (Stack)       │
│  ┌───────┬───────┐      │
│  │  10   │  20   │      │  ← 所有数据都在栈上
│  └───────┴───────┘      │     可以直接位复制
│     t1.0   t1.1         │
└─────────────────────────┘

赋值时：t2 = t1
┌─────────────────────────┐
│         栈 (Stack)       │
│  ┌───────┬───────┐      │
│  │  10   │  20   │  ← t1│
│  ├───────┼───────┤      │
│  │  10   │  20   │  ← t2│  ← 简单的位复制
│  └───────┴───────┘      │
└─────────────────────────┘

两个元组完全独立，互不影响
```

#### 数组的 Copy 规则

数组（Array）是否 Copy **同样取决于其元素类型**：

```rust
// ✅ 这些数组是 Copy 的
let a1 = [1, 2, 3];           // [i32; 3]
let a2 = [1.5, 2.5, 3.5];     // [f64; 3]
let a3 = ['a', 'b', 'c'];     // [char; 3]
let a4 = [true, false];       // [bool; 2]

let x = a1;
println!("{:?}", a1);  // ✅ a1 仍然有效
```

```rust
// ❌ 这些数组 NOT Copy
let a5 = [String::from("a"), String::from("b")];  // [String; 2]
let a6 = [vec![1, 2], vec![3, 4]];                // [Vec<i32>; 2]

let y = a5;
// println!("{:?}", a5);  // ❌ 错误：a5 已移动
```

**内存布局图解：**

```
Copy 数组 [i32; 3]：
┌─────────────────────────────────┐
│         栈 (Stack)               │
│  ┌──────┬──────┬──────┐        │
│  │  1   │  2   │  3   │  ← a1  │
│  └──────┴──────┴──────┘        │
└─────────────────────────────────┘

赋值时：a2 = a1
┌─────────────────────────────────┐
│         栈 (Stack)               │
│  ┌──────┬──────┬──────┐  ← a1  │
│  │  1   │  2   │  3   │        │
│  ├──────┼──────┼──────┤        │
│  │  1   │  2   │  3   │  ← a2  │
│  └──────┴──────┴──────┘        │
└─────────────────────────────────┘

整个数组在栈上，直接位复制
```

#### 为什么这个规则很重要？

```
如果元组/数组包含非 Copy 类型（如 String）：

┌─────────────────────────────────────┐
│  栈 (Stack)                          │
│  ┌──────────┬──────────┐            │
│  │  ptr     │   42     │  ← t1      │
│  │  ↓       │          │            │
│  └──────────┴──────────┘            │
│       │                             │
│       ↓                             │
│  ┌──────────────┐                   │
│  │  堆 (Heap)    │                   │
│  │  "hello"     │                   │
│  └──────────────┘                   │
└─────────────────────────────────────┘

如果自动 Copy：
- t2 会复制指针，但不复制堆数据
- 两个元组指向同一块堆内存
- 当两者都离开作用域时，会双重释放！

Rust 的选择：
- 如果包含非 Copy 类型，整个元组/数组就不是 Copy
- 赋值时执行移动语义，原变量失效
- 避免双重释放问题
```

#### 官方定义验证

在 Rust 标准库中，元组和数组的 Copy 实现是这样的：

```rust
// 元组的 Copy 实现（简化）
impl<T1, T2> Copy for (T1, T2)
where
    T1: Copy,
    T2: Copy,
{}

// 数组的 Copy 实现（简化）
impl<T, const N: usize> Copy for [T; N]
where
    T: Copy,
{}
```

#### Copy 类型总结表

| 类型 | 是否 Copy | 条件 |
|------|----------|------|
| `(i32, i32)` | ✅ | 所有元素 Copy |
| `(i32, f64, bool)` | ✅ | 所有元素 Copy |
| `(String, i32)` | ❌ | String 不是 Copy |
| `[i32; 5]` | ✅ | 元素 i32 是 Copy |
| `[f64; 10]` | ✅ | 元素 f64 是 Copy |
| `[String; 3]` | ❌ | String 不是 Copy |
| `[[i32; 3]; 4]` | ✅ | 嵌套数组元素都是 Copy |

**核心规则**：元组和数组本身不决定 Copy，而是由其**元素类型**决定。只有当所有元素都是 Copy 时，整个元组或数组才是 Copy。

### 不是 Copy 的类型

```rust
fn main() {
    // String - 不是 Copy
    let s1 = String::from("hello");
    let s2 = s1;
    // println!("{}", s1);  // ❌ 错误

    // Vec - 不是 Copy
    let v1 = vec![1, 2, 3];
    let v2 = v1;
    // println!("{:?}", v1);  // ❌ 错误

    // Box - 不是 Copy
    let b1 = Box::new(42);
    let b2 = b1;
    // println!("{:?}", b1);  // ❌ 错误

    // 包含非 Copy 类型的元组 - 不是 Copy
    let t1 = (String::from("hello"), 5);
    let t2 = t1;
    // println!("{:?}", t1);  // ❌ 错误
}
```

### Copy vs Clone

```
┌─────────────────────────────────────────────────────┐
│              Copy vs Clone                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Copy                                               │
│  ├── 编译时自动复制                                 │
│  ├── 只复制栈数据（位复制）                         │
│  ├── 零开销                                         │
│  └── 适用于简单类型（整数、浮点、bool、char）        │
│                                                     │
│  Clone                                              │
│  ├── 显式调用 .clone()                              │
│  ├── 可能复制堆数据（深拷贝）                       │
│  ├── 可能有性能开销                                 │
│  └── 适用于复杂类型（String、Vec、自定义类型）       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 7.8 完整示例

### 示例 1：所有权传递链

```rust
fn main() {
    // 创建所有权链
    let s1 = String::from("start");

fn pass_to_function(s: String) -> String {
    println!("收到：{}", s);

}

    s  // 返回修改后的所有权
}

收到：start
最终结果：start - modified!
```

fn main() {

## 7.9 常见错误

### 错误 1：使用已移动的值

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;

    // println!("{}", s1);  // ❌ 错误：使用了已移动的值
    println!("{}", s2);  // ✅ 正确
}
```

**错误信息：**
```
error[E0382]: use of moved value: `s1`
 --> src/main.rs:5:20
  |
3 |     let s1 = String::from("hello");
  |         -- move occurs because `s1` has type `String`
4 |     let s2 = s1;
  |              -- value moved here
5 |     println!("{}", s1);
  |                    ^^ value used here after move
```

### 错误 2：函数参数移动后继续使用

```rust
fn print_string(s: String) {
    println!("{}", s);
}

fn main() {
    let s = String::from("hello");
    print_string(s);

    // println!("{}", s);  // ❌ 错误：s 已移动
}
```

**解决方案：**

```rust
// 方案 1：传递引用
fn print_string(s: &String) {
    println!("{}", s);
}

fn main() {
    let s = String::from("hello");
    print_string(&s);
    println!("{}", s);  // ✅ s 仍然有效
}

// 方案 2：接受&str（更佳）
fn print_string(s: &str) {
    println!("{}", s);
}

fn main() {
    let s = String::from("hello");
    print_string(&s);  // &String → &str
    print_string("literal");  // &'static str
}

// 方案 3：使用 clone
fn print_string(s: String) {
    println!("{}", s);
}

fn main() {
    let s = String::from("hello");
    print_string(s.clone());  // 复制一份
    println!("{}", s);  // ✅ s 仍然有效
}
```

### 错误 3：在循环中移动

```rust
fn main() {
    let s = String::from("hello");

    // ❌ 错误：试图在循环中多次使用 s
    // for _ in 0..3 {
    //     print_string(s);  // 第一次后 s 就无效了
    // }
}

fn print_string(s: String) {
    println!("{}", s);
}
```

**解决方案：**

```rust
// 方案 1：传递引用
fn main() {
    let s = String::from("hello");

    for _ in 0..3 {
        print_string(&s);
    }
}

fn print_string(s: &str) {
    println!("{}", s);
}

// 方案 2：使用 clone
fn main() {
    let s = String::from("hello");

    for _ in 0..3 {
        print_string(s.clone());
    }
    // s 仍然可用
    println!("{}", s);
}
```

## 7.12 小结

### 所有权核心规则

| 规则 | 说明 |
|------|------|
| 唯一所有 | 每个值只有一个所有者 |
| 移动语义 | 赋值转移所有权，原变量无效 |
| 作用域释放 | 所有者离开作用域，值被 drop |

### Copy trait

| 类型 | Copy? | 说明 |
|------|-------|------|
| 整数 (i8-u128, isize, usize) | ✅ | 位复制 |
| 浮点数 (f32, f64) | ✅ | 位复制 |
| 布尔 (bool) | ✅ | 位复制 |
| 字符 (char) | ✅ | 位复制 |
| 元组（元素全 Copy） | ✅ | 位复制 |
| 数组（元素全 Copy） | ✅ | 位复制 |
| String, Vec, Box | ❌ | 需要 clone() |

### 关键要点

1. **所有权是 Rust 内存安全的核心**
2. **移动是零成本的（只复制栈数据）**
3. **需要深拷贝时显式调用 `clone()`**
4. **函数参数默认移动所有权**
5. **Copy 类型自动复制，不使用移动语义**
