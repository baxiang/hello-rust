## 20.2 模块系统基础

### 为什么需要模块

```rust
// 没有模块：所有代码混在一起
fn process_data() {}
fn save_file() {}
fn send_email() {}
fn validate_user() {}
// ... 1000 行后
fn process_data() {}  // ❌ 命名冲突！

// 有模块：代码组织清晰
mod data {
    pub fn process() {}
}

mod file {
    pub fn save() {}
}

mod email {
    pub fn send() {}
}

mod user {
    pub fn validate() {}
}
```

### 定义模块

```rust
// 方式 1：内联定义（适合小模块）
mod front_of_house {
    pub mod hosting {
        pub fn add_to_waitlist() {
            println!("Added to waitlist");
        }

        pub fn seat_at_table(table_id: usize) -> usize {
            println!("Seating at table {}", table_id);
            table_id
        }
    }

    mod serving {
        fn take_order() {}
        fn serve_order() {}
    }
}

fn main() {
    // 使用绝对路径
    front_of_house::hosting::add_to_waitlist();

    // 使用相对路径
    crate::front_of_house::hosting::add_to_waitlist();
}
```

### 模块树结构

```
crate（根）
└── front_of_house
    ├── hosting
    │   ├── add_to_waitlist
    │   └── seat_at_table
    └── serving
        ├── take_order
        └── serve_order

关键点：
• 模块可以嵌套
• pub 控制可见性
• 默认所有项都是私有的
```
