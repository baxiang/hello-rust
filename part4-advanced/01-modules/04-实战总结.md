## 20.8 Cargo.toml 配置

### 基本配置

```toml
[package]
name = "my_package"
version = "0.1.0"
edition = "2021"
authors = ["Your Name <you@example.com>"]
description = "A brief description"
license = "MIT"
repository = "https://github.com/user/repo"
keywords = ["keyword1", "keyword2"]
categories = ["development-tools"]
```

### 定义多个二进制

```toml
[[bin]]
name = "my_tool"
path = "src/bin/my_tool.rs"

[[bin]]
name = "my_server"
path = "src/bin/my_server.rs"
```

### 库配置

```toml
[lib]
name = "my_lib"
path = "src/lib.rs"
crate-type = ["lib", "rlib", "cdylib"]
```






## 20.9 常见错误

### 错误 1：模块文件找不到

```
error[E0583]: file not found for module `xyz`
 --> src/lib.rs:1:1
  |
1 | mod xyz;
  | ^^^^^^^^

原因：缺少 src/xyz.rs 或 src/xyz/mod.rs
解决：创建对应的模块文件
```

### 错误 2：私有模块无法访问

```rust
mod private {
    pub fn func() {}
}

fn main() {
    // private::func();  // ❌ 错误：模块是私有的
}

// 修复
pub mod public {
    pub fn func() {}
}

fn main() {
    public::func();  // ✅
}
```

### 错误 3：路径错误

```rust
// ❌ 错误
// use non_existent::module;

// ✅ 正确
use crate::existent::module;
```

### 错误 4：循环依赖

```rust
// ❌ 错误
mod a {
    use crate::b;  // a 使用 b
}

mod b {
    use crate::a;  // b 使用 a，循环依赖
}

// 解决：重构代码，避免循环依赖
```






## 20.10 练习

### 练习 1：基本模块

创建一个库项目，包含以下模块：
- `math`：包含 `add`, `subtract`, `multiply`, `divide` 函数
- `string`：包含 `reverse`, `uppercase`, `lowercase` 函数

### 练习 2：模块文件分离

将练习 1 的代码分离到多个文件中：
```
src/
├── lib.rs
├── math.rs
└── string.rs
```

### 练习 3：重新导出

使用 `pub use`重新导出`math::add`和`math::subtract`，使得外部可以直接使用`my_crate::add`。

### 模块系统速查表

| 语法 | 说明 |
|------|------|
| `mod foo;` | 声明模块 |
| `pub mod foo;` | 声明公共模块 |
| `use foo::bar;` | 引入模块 |
| `pub use foo::bar;` | 重新导出 |
| `crate::` | crate 根 |
| `super::` | 父模块 |
| `self::` | 当前模块 |
