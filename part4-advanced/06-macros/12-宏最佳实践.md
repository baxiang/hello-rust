## 25.12 宏最佳实践

```
┌─────────────────────────────────────────────────────┐
│              宏最佳实践                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 优先使用函数                                     │
│     • 函数更容易理解和调试                           │
│     • 只在必要时使用宏                               │
│                                                     │
│  2. 遵循 Rust 语法                                   │
│     • 让宏调用看起来像普通 Rust 代码                  │
│     • 使用熟悉的语法结构                           │
│                                                     │
│  3. 提供良好文档                                     │
│     • 说明宏的用法                                   │
│     • 提供示例                                       │
│                                                     │
│  4. 注意卫生                                         │
│     • 避免变量名冲突                                 │
│     • 使用唯一内部变量名                             │
│                                                     │
│  5. 测试宏                                           │
│     • 测试各种输入情况                               │
│     • 测试边界情况                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```
