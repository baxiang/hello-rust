## 1.4 运行第一个程序

### 方式一：cargo run（推荐）

```bash
cargo run
```

输出：

```
   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished dev [unoptimized + debuginfo] target(s) in 0.23s
     Running `target/debug/hello_rust`
Hello, Rust!
```

**输出解读：**

- `Compiling`：正在编译
- `dev`：调试模式（未优化，包含调试信息）
- `Finished`：编译完成
- `Running`：正在运行

### 方式二：先构建，再运行

```bash
# 构建
cargo build

# 运行生成的二进制文件
./target/debug/hello_rust
```

### 调试模式 vs 发布模式

```bash
# 调试模式（开发用）
cargo build
# 输出：target/debug/hello_rust
# 特点：编译快，运行慢，有调试信息

# 发布模式（生产用）
cargo build --release
# 输出：target/release/hello_rust
# 特点：编译慢，运行快，无调试信息
```

性能对比：

```bash
# 查看文件大小
ls -lh target/debug/hello_rust
ls -lh target/release/hello_rust
```

通常 release 版本：
- 文件更小
- 运行快 10-100 倍






## 1.6 动手实践

### 练习 1：修改程序

编辑 `src/main.rs`：

```rust
fn main() {
    println!("你好，Rust!");
    println!("这是第二行");
    println!("第三行...");
}
```

运行：

```bash
cargo run
```

### 练习 2：查看编译输出

```bash
# 查看详细编译过程
cargo build -v

# 查看生成的二进制文件
file target/debug/hello_rust

# 查看文件大小
ls -lh target/debug/hello_rust
```

### 练习 3：创建库项目

```bash
# 创建库项目
cargo new my_lib --lib
cd my_lib
ls -la
```

库项目结构：

```
my_lib/
├── Cargo.toml
└── src/
    └── lib.rs    # 注意：是 lib.rs 不是 main.rs
```

编辑 `src/lib.rs`：

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }
}
```

运行测试：

```bash
cargo test
```
