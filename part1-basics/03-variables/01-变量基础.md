## 03.1 变量声明

### let 关键字

```rust
fn main() {
    // 声明一个变量
    let x = 5;
    println!("x = {}", x);

    // 变量默认是不可变的（immutable）
    // x = 6;  // ❌ 错误：不能给不可变变量重新赋值
}
```

### 变量命名规则

```rust
fn main() {
    // 推荐使用 snake_case（蛇形命名法）
    let my_variable = 10;
    let count = 20;

    // 可以使用下划线开头（表示暂未使用）
    let _unused = 5;

    // 可以覆盖外部变量（shadowing）
    let x = 5;
    let x = 10;  // 这是允许的
    println!("x = {}", x);  // 输出 10
}
```

## 03.2 可变性

### mut 关键字

```rust
fn main() {
    // 使用 mut 声明可变变量
    let mut y = 5;
    println!("y = {}", y);

    y = 10;  // ✅ 正确：y 是可变的
    println!("y = {}", y);

    // 不可变 vs 可变
    let a = 5;      // 不可变
    let mut b = 5;  // 可变

    // a = 6;       // ❌ 错误
    b = 10;         // ✅ 正确
}
```

### 为什么默认不可变

```
┌─────────────────────────────────────────────────────┐
│          为什么变量默认不可变？                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 安全性                                           │
│     • 防止意外修改                                   │
│     • 减少 bug                                       │
│                                                     │
│  2. 并发安全                                         │
│     • 不可变数据天然线程安全                        │
│     • 编译器可以做得更好优化                         │
│                                                     │
│  3. 代码可读性                                       │
│     •  reader 知道值不会改变                         │
│     • 更容易推理代码行为                             │
│                                                     │
│  Rust 哲学：让正确的事情变得容易                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```
