## 5.1 为什么需要函数？

### 函数的作用

想象你在组装家具：

```
没有函数：
┌─────────────────────────────────────────┐
│ 1. 拿螺丝刀                              │
│ 2. 拿起螺丝                             │
│ 3. 拧螺丝...重复 100 次                  │
│ 4. 拿锤子                               │
│ 5. 敲钉子...重复 50 次                   │
│ ...代码重复，难以维护...                 │
└─────────────────────────────────────────┘

有函数：
┌─────────────────────────────────────────┐
│ fn 拧螺丝 (螺丝，位置) { ... }          │
│ fn 敲钉子 (钉子，位置) { ... }          │
│                                         │
│ fn 组装 () {                            │
│     拧螺丝 (螺丝 1, 位置 A);             │
│     敲钉子 (钉子 1, 位置 B);             │
│ }                                       │
│                                         │
│ → 代码复用，易于理解和维护              │
└─────────────────────────────────────────┘
```

### 函数的优势

| 优势 | 说明 |
|------|------|
| 代码复用 | 写一次，用多次 |
| 模块化 | 将大问题分解为小问题 |
| 抽象 | 隐藏实现细节，只暴露接口 |
| 可测试 | 独立测试每个函数 |
| 可维护 | 修改一处，影响全局 |
