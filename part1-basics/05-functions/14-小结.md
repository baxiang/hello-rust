## 5.14 小结

### 函数语法要点

| 概念 | 语法 | 说明 |
|------|------|------|
| 函数定义 | `fn name(params) -> Type { body }` | 基本结构 |
| 参数类型 | `x: i32` | 必须显式标注 |
| 返回类型 | `-> i32` | 有返回值时必须 |
| 表达式返回 | `x + y` | 无分号 |
| 语句返回 | `return x + y` | 显式返回或提前返回 |
| 单元类型 | `()` | 无返回值时的隐含返回 |

### 关键概念

1. **表达式 vs 语句**
   - 表达式返回值（无分号）
   - 语句执行操作（有分号）

2. **所有权传递**
   - 传递值：转移所有权
   - 传递引用：借用

3. **参数命名**
   - 使用蛇形命名（snake_case）
   - 选择有意义的名字

### 最佳实践

1. 函数应该短小，只做一件事
2. 函数名应该清楚表达其功能
3. 优先使用 `&str` 而不是 `&String`
4. 使用 `Option` 或 `Result` 处理可能的失败
5. 写文档注释说明函数的用途
