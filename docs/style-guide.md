# Rust 教程文档风格指南

> 确保文档质量和一致性

---

## 核心原则

### 1. 简洁清晰

- 使用简单直接的语言
- 避免冗长复杂的句子
- 一个段落表达一个观点

### 2. 代码优先

- 每个概念都提供代码示例
- 标注正确和错误示例
- 提供运行结果

### 3. 可视化辅助

- 使用 ASCII 艺术绘制图解
- 展示内存布局和流程
- 增强理解效果

### 4. 互动学习

- 每章包含练习题
- 提供答案和解释
- 鼓励动手实践

## Markdown 格式规范

### 标题层级

```markdown
# 章节标题（一级，每个文件只有一个）
> 章节简介（引用块）

## 主要节（二级，主要内容分节）
### 子节（三级，详细内容）
```

**规则：**
- 不使用四级以下标题
- 标题层级不超过三级
- 标题简洁明确

---

### 引用块使用

用于章节开头简介：

```markdown
# 章节名

> 简短介绍章节内容和目标

## 正文内容
```

用于重要提示：

```markdown
> ⚠️ 重要提示：注意事项
```

用于代码说明：

```markdown
> 💡 提示：代码解释
```

## ASCII 艺术规范

### 框线风格

使用标准 ASCII 字符：

```
┌ ─ ┐  顶部
│   │  侧边
└ ─ ┘  底部
├ ─ ┤  分隔
┬ ┴    连接
```

### 示例

```markdown
\`\`\`
┌─────────────────┐
│  内存布局图     │
├─────────────────┤
│  栈内存         │
│  堆内存         │
└─────────────────┘
\`\`\`
```

### 流程图

```markdown
\`\`\`
流程 A → 流程 B → 流程 C

详细流程：
1. 输入数据
   ↓
2. 处理数据
   ↓
3. 输出结果
\`\`\`
```

## 代码示例规范

### Rust 代码风格

遵循 AGENTS.md 中的规范：

1. **格式化：** 使用 `cargo fmt`
2. **类型标注：** 公开 API 必须标注
3. **错误处理：** 使用 Result，避免 panic
4. **命名规范：** snake_case 函数，PascalCase 类型

### 示例代码结构

```rust
// ✅ 正确示例
/// 函数说明
fn good_example() {
    // 清晰的代码
}

// ❌ 错误示例
// 错误代码，附带错误信息
fn bad_example() {
    // 错误的实现
}
```

**运行：**

```bash
$ cargo run
[输出结果]
```
```

## 中文术语对照

保留英文原文对照：

| 中文 | 英文 | 使用场景 |
|------|------|----------|
| 所有权 | Ownership | 解释概念时用中文，代码注释用英文 |
| 借用 | Borrowing | 同上 |
| 生命周期 | Lifetime | 同上 |
| Trait | Trait | **保留英文** |
| Crate | Crate | **保留英文** |
| Module | Module | 可用"模块" |
| 泛型 | Generics | 可用中文 |
| 闭包 | Closure | 可用中文 |

**规则：**
- 正文中可使用中文术语
- 代码中保留英文术语
- 特殊术语（Trait, Crate）始终保留英文
