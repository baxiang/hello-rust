### 2. 场景驱动教学

每个概念必须从"为什么需要"开始：

| 传统写法 | 本教程写法 |
|---------|-----------|
| "所有权是 Rust 的核心特性" | "C++ 程序泄漏内存怎么办？所有权解决这个问题" |
| "Option 表示可能无值" | "Java 的 null 导致 NPE，Rust 用 Option 避免" |
| "&mut 是可变引用" | "函数需要修改参数值，但又不想转移所有权" |

### 3. 对比学习法

与已知语言对比，降低认知负担：

```
┌─────────────────────────────────────────────────────┐
│              语言对比学习                             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Java/C# 程序员：                                    │
│  ├── null → Option<T>（必须处理无值）              │
│  ├── 引用 → &T（借用，不拥有）                      │
│  ├── 异常 → Result<T, E>（显式处理错误）            │
│                                                     │
│  Python 程序员：                                     │
│  ├── list → Vec<T>（类型固定）                     │
│  ├── dict → HashMap<K, V>                          │
│  ├── 函数返回多值 → 元组/结构体                     │
│                                                     │
│  C/C++ 程序员：                                      │
│  ├── 指针 → 引用（编译器保证安全）                  │
│  ├── 手动管理 → 所有权自动管理                     │
│  ├── 内存泄漏 → 编译器阻止                          │
│                                                     │
│  Go 程序员：                                          │
│  ├── GC → 无 GC（零运行时开销）                     │
│  ├── goroutine → async/task                        │
│  ├── interface → Trait（更严格）                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4. 编译器当导师

教会读懂错误信息，编译器是最好的老师：

**错误诊断模板（每章必须包含）：**

```markdown
## 常见错误诊断

### 错误类型：使用已移动的值

**错误代码：**
```rust
let s1 = String::from("hello");
let s2 = s1;
println!("{}", s1);  // ❌
```

**编译器信息解读：**
```
error[E0382]: borrow of moved value: `s1`
  │ 说明：s1 已被移动，不能再借用
  │ 提示：编译器告诉你错误原因
```

**修复方案：**
```rust
// 方案 1：克隆
let s2 = s1.clone();

// 方案 2：使用引用
let s2 = &s1;

// 方案 3：返回所有权
let s2 = take_and_return(s1);
```

**关键点：编译器信息是诊断线索，不是障碍**
```

## Rustdoc 文档注释规范

Rust 的文档注释（`///`）支持 doctest，是教程编写的重要规范。

### 文档注释格式

```rust
/// 函数功能简述（一句话）
///
/// 详细说明（可选）：
/// - 参数说明
/// - 返回值说明
/// - 错误情况
///
/// # 示例
///
/// ```
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
///
/// # 错误
///
/// 如果输入为空，返回 `Err(InvalidInput)`
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

### 示例代码块要求

| 类型 | 格式 | 说明 |
|------|------|------|
| 可测试示例 | ` ``` ` | 无语言标记或 `rust`，会被 doctest 运行 |
| 不可测试 | ` ```rust,ignore ` | 需要 IO、网络等无法在 doctest 运行的场景 |
| 应失败示例 | ` ```rust,should_panic ` | 演示 panic 场景 |
| 编译失败 | ` ```rust,compile_fail ` | 演示编译错误 |

## 常见 Rust 惯用法

教程中的代码示例应优先使用 Rust 惯用写法。

### 模式匹配风格

```rust
// ✅ 推荐：使用 if let 简化单分支
if let Some(value) = option {
    println!("{}", value);
}

// ✅ 推荐：使用 match 处理多分支
match result {
    Ok(value) => println!("{}", value),
    Err(e) => eprintln!("Error: {}", e),
}

// ❌ 避免：过度嵌套
if option.is_some() {
    let value = option.unwrap();
    println!("{}", value);
}
```

### Iterator 链式调用

```rust
// ✅ 推荐：链式调用
let sum: i32 = numbers.iter()
    .filter(|&&x| x > 0)
    .map(|&x| x * 2)
    .sum();

// ❌ 避免：临时变量过多
let mut result = 0;
for &x in &numbers {
    if x > 0 {
        result += x * 2;
    }
}
```

### 错误传播

```rust
// ✅ 推荐：使用 ? 操作符
fn read_file(path: &str) -> Result<String, io::Error> {
    let mut contents = String::new();
    File::open(path)?.read_to_string(&mut contents)?;
    Ok(contents)
}

// ❌ 避免：手动 match
fn read_file(path: &str) -> Result<String, io::Error> {
    let file = match File::open(path) {
        Ok(f) => f,
        Err(e) => return Err(e),
    };
    // ...
}
```

### Option 处理

```rust
// ✅ 推荐：使用组合子
let value = config.get("key").unwrap_or("default");
let doubled = number.map(|x| x * 2);

// ✅ 推荐：使用 if let
if let Some(user) = find_user(id) {
    process(user);
}
```

### 所有权转移与借用

```rust
// ✅ 推荐：不必要时不转移所有权
fn process(data: &[u8]) {
    // 使用切片，不获取所有权
}

// ✅ 推荐：需要修改时使用 &mut
fn update(list: &mut Vec<i32>) {
    list.push(42);
}
```

### 关键代码解释格式

**何时需要解释？**

| 情况 | 是否需要 | 处理方式 |
|------|---------|---------|
| 最简示例（1-3 行） | ❌ 不需要 | 代码本身已足够清晰 |
| 新语法概念（生命周期、泛型） | ✅ 必须解释 | 用表格或注释说明 |
| 所有权相关代码 | ✅ 必须解释 | 标注所有权转移点 |
| 复杂逻辑（多步骤） | ✅ 必须解释 | 分步骤注释 |
| 嵌套/组合结构 | ✅ 必须解释 | 拆解结构说明 |

**表格格式示例：**

```markdown
```rust
fn first_or_default<T: Default>(items: &[T]) -> T {
    items.first().cloned().unwrap_or_default()
}
```

**关键代码说明：**

| 代码片段 | 含义 | 为什么这样写 |
|----------|------|-------------|
| `&[T]` | 切片引用 | 不获取所有权，避免移动 |
| `.first()` | 取第一个元素 | 返回 `Option<&T>` |
| `.cloned()` | 克隆元素 | 将 `&T` 变为 `T`，需要 `Clone` |
| `unwrap_or_default()` | 空则返回默认值 | 需要 `Default` trait |
```

**注释格式示例：**

```markdown
```rust
fn process(s: &str) -> &str {
    // 第一步：查找边界
    // ↓ s 是借用，不获取所有权
    let bytes = s.as_bytes();
    
    // 第二步：处理数据
    // ↓ 返回值生命周期与 s 相同
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];  // 切片，所有权不变
        }
    }
    
    &s[..]
}
```
```

## 中文术语对照

保留英文原文对照：

| 中文 | 英文 | 使用场景 |
|------|------|----------|
| 所有权 | Ownership | 解释概念时用中文，代码注释用英文 |
| 借用 | Borrowing | 同上 |
| 生命周期 | Lifetime | 同上 |
| Trait | Trait | **保留英文** |
| Crate | Crate | **保留英文** |
| Module | Module | 可用"模块" |
| 泛型 | Generics | 可用中文 |
| 闭包 | Closure | 可用中文 |

**规则：**
- 正文中可使用中文术语
- 代码中保留英文术语
- 特殊术语（Trait, Crate）始终保留英文
