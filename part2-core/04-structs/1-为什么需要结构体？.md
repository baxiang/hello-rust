## 10.1 为什么需要结构体？

### 问题场景

```rust
// 问题：描述一个人
// 使用多个独立变量（混乱）
let name1 = String::from("Alice");
let age1 = 25;
let email1 = String::from("alice@example.com");

let name2 = String::from("Bob");
let age2 = 30;
let email2 = String::from("bob@example.com");

// 问题：
// 1. 数据分散，不直观
// 2. 难以传递给函数
// 3. 容易搞混（name1 对应 age1 吗？）
```

### 结构体的解决方案

```rust
// 使用结构体（清晰）
struct Person {
    name: String,
    age: u32,
    email: String,
}

let person1 = Person {
    name: String::from("Alice"),
    age: 25,
    email: String::from("alice@example.com"),
};

let person2 = Person {
    name: String::from("Bob"),
    age: 30,
    email: String::from("bob@example.com"),
};

// 优势：
// 1. 数据组织在一起，语义清晰
// 2. 容易传递和使用
// 3. 类型安全
```

### 结构体 vs 元组

```
┌─────────────────────────────────────────────────────┐
│           结构体 vs 元组                             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  元组 (Tuple)                                       │
│  ├── 字段无名字（.0, .1, .2）                       │
│  ├── 适合临时组合                                   │
│  └── 例：(x, y) = 坐标                              │
│                                                     │
│  结构体 (Struct)                                    │
│  ├── 字段有名字（.name, .age）                      │
│  ├── 适合描述现实事物                               │
│  └── 例：Person { name, age, email }               │
│                                                     │
└─────────────────────────────────────────────────────┘
```
