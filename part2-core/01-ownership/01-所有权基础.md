# 为什么需要所有权？

> Rust 最独特的特性，内存安全的核心保障——无需垃圾回收器

---

## 其他语言的内存管理方式

```
┌─────────────────────────────────────────────────────┐
│              内存管理方式对比                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  C/C++：手动管理                                     │
│  ├── 程序员负责分配/释放                            │
│  ├── 灵活但容易出错                                 │
│  └── 常见问题：内存泄漏、悬垂指针、双重释放          │
│                                                     │
│  Java/Python/Go：垃圾回收（GC）                      │
│  ├── 运行时自动回收                                 │
│  ├── 安全但有性能开销                               │
│  └── 问题：GC 暂停、不可预测的延迟                   │
│                                                     │
│  Rust：所有权系统                                    │
│  ├── 编译时检查                                     │
│  ├── 零运行时开销                                   │
│  └── 安全且高效！                                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## 所有权解决的问题

```rust
// C 风格：内存泄漏
void create_leak() {
    char* buffer = malloc(100);
    // 忘记 free，内存泄漏！
}

// Rust：编译时自动释放
fn no_leak() {
    let buffer = String::with_capacity(100);
    // 离开作用域时自动释放！
}
```

# 所有权的三大规则

> Rust 内存管理的核心定律

---

## 所有权三定律

```
┌─────────────────────────────────────────────────────┐
│              所有权三定律                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  第一定律：每个值都有一个所有者                     │
│           Each value has an owner                  │
│                                                     │
│  第二定律：任一时刻只能有一个所有者                │
│           There can only be one owner at a time    │
│                                                     │
│  第三定律：当所有者离开作用域，值会被丢弃          │
│           When the owner goes out of scope,        │
│           the value will be dropped                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## 规则详解

```rust
fn main() {
    // 第一定律：每个值都有一个所有者
    let s = String::from("hello");  // s 是 "hello" 的所有者

    {
        // 第二定律：任一时刻只能有一个所有者
        let t = s;  // 所有权从 s 移动到 t
        // 现在 t 是所有者，s 不再有效

        println!("{}", t);  // ✅ 正确
        // println!("{}", s);  // ❌ 错误：s 已无效
    }
    // t 离开作用域

    // 第三定律：所有者离开作用域，值被丢弃
    let u = String::from("world");
}  // u 离开作用域，"world" 被自动释放
```

---

## 规则记忆口诀

```
一值一主，时刻唯一
主离作用域，值即丢弃
```

# 作用域详解

> 理解变量的生命周期

---

## 作用域基础

```rust
fn main() {
    // s 从这里开始有效
    let s = String::from("hello");

    println!("{}", s);  // 可以使用 s
}  // s 离开作用域，内存被自动释放
   // Rust 在这里隐式调用 drop(s)
```

---

## 作用域嵌套

```rust
fn main() {
    let outer = String::from("外层");

    {
        let inner = String::from("内层");
        println!("inner: {}", inner);  // ✅ 可以
        println!("outer: {}", outer);  // ✅ 也可以
    }  // inner 离开作用域，被释放

    println!("outer: {}", outer);  // ✅ outer 仍然有效
    // println!("inner: {}", inner);  // ❌ inner 已无效
}  // outer 离开作用域，被释放
```

---

## 作用域与内存释放时机

```rust
fn main() {
    let s1 = String::from("first");
    let s2 = String::from("second");
    let s3 = String::from("third");

    // s1, s2, s3 都在作用域中

    {
        let s4 = String::from("fourth");
        println!("s4: {}", s4);
    }  // s4 在这里被释放

    println!("s1: {}", s1);
    println!("s2: {}", s2);
    println!("s3: {}", s3);
}  // s3, s2, s1 按相反顺序被释放
   // 释放顺序：s3 → s2 → s1（后进先出）
```

---

## 作用域规则总结

| 规则 | 说明 |
|------|------|
| 进入作用域 | 变量开始有效 |
| 离开作用域 | 变量被释放 |
| 嵌套作用域 | 内层可访问外层，外层不能访问内层 |
| 释放顺序 | 后进先出（LIFO） |
