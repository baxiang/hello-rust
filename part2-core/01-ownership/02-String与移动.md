# String 类型详解

> String 与 &str 的区别


## String vs &str

这是 Rust 初学者最容易困惑的地方，我们详细对比：

```
┌─────────────────────────────────────────────────────┐
│           String vs &str 详细对比                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  String（拥有所有权的字符串）                        │
│  ├── 可增长、可修改                                  │
│  ├── 数据存储在堆上                                 │
│  ├── 大小：24 字节（指针 + 长度 + 容量）              │
│  ├── 创建：String::from("hello")                    │
│  └── 所有权转移时会影响性能（移动）                  │
│                                                     │
│  &str（字符串切片/视图）                            │
│  ├── 不可变（通常）                                  │
│  ├── 借用数据，不拥有所有权                         │
│  ├── 大小：16 字节（指针 + 长度）                     │
│  ├── 字面量："hello"（类型是 &'static str）          │
│  └── 传递时只复制指针，非常高效                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```


## 内存布局可视化

```
字符串字面量 &str：
┌─────────────────────────────────────────┐
│           栈（Stack）                    │
│  ┌───────────────┬───────────────┐      │
│  │    指针       │     长度      │      │
│  │  (指向静态区) │    (5)        │      │
│  └───────┬───────┴───────────────┘      │
│          │                               │
│          ▼                               │
│  ┌─────────────────────────────────┐    │
│  │       静态区（编译时常量）        │    │
│  │  "hello\0"                      │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘

String（堆分配字符串）：
┌─────────────────────────────────────────┐
│           栈（Stack）                    │
│  ┌───────────┬───────────┬─────────┐   │
│  │   指针    │   长度    │  容量   │   │
│  │ (指向堆)  │   (5)     │  (10)   │   │
│  └─────┬─────┴───────────┴─────────┘   │
│        │                                │
│        ▼                                │
│  ┌─────────────────────────────────┐   │
│  │       堆（Heap）                 │   │
│  │  "hello"  │  空闲空间 (5)       │   │
│  └─────────────────────────────────┘   │
│                                         │
│  可以增长：push_str(" world")           │
└─────────────────────────────────────────┘
```


## 代码示例

```rust
fn main() {
    // &str - 字符串字面量
    let s1: &str = "hello";

    // String - 可增长字符串
    let s2: String = String::from("hello");
    let s3: String = "hello".to_string();
    let s4: String = "hello".to_owned();

    // 修改 String
    let mut s5 = String::from("hello");
    s5.push_str(" world!");
    println!("{}", s5);  // hello world!

    // &str 不能修改
    // s1.push_str(" world");  // ❌ 错误

    // 类型验证
    println!("s1 类型：{:?}", std::any::type_name_of_val(&s1));
    println!("s2 类型：{:?}", std::any::type_name_of_val(&s2));
}
```


## 选择建议

| 场景 | 推荐 |
|------|------|
| 函数参数 | `&str`（更通用） |
| 结构体字段 | `String`（拥有所有权） |
| 字符串字面量 | `&'static str` |
| 需要修改 | `String` |

# 移动语义

> 理解所有权的转移


## 移动 vs 复制

```rust
fn main() {
    // 对于 String 类型：移动语义
    let s1 = String::from("hello");
    let s2 = s1;  // 移动所有权

    // println!("{}", s1);  // ❌ 错误：s1 已无效
    println!("{}", s2);  // ✅ 正确

    // 对于整数类型：复制语义（Copy trait）
    let x = 5;
    let y = x;  // 复制值

    println!("x = {}, y = {}", x, y);  // 都有效
}
```


## 移动语义图解

```
移动前：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
└──────┘            └─────────┘

let s2 = s1;  之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ (无效)     │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            │         │
│ s2   │ ───────────┘         │
│ ptr  │                      │
│ len  │                      │
│ cap  │                      │
└──────┘                      │

关键点：
• s2 获得了指向堆数据的指针
• s1 被标记为无效（不能使用）
• 数据没有被复制！
• 当 s2 离开作用域时，数据被释放
```


## 为什么 Rust 不自动深拷贝？

```
假设 Rust 自动深拷贝：
let s1 = String::from("hello");
let s2 = s1;  // 如果自动复制堆数据...

问题：
┌─────────────────────────────────────────┐
│ 1. 性能问题                              │
│    - 堆数据可能很大（几 MB 甚至几 GB）     │
│    - 每次赋值都要复制，非常慢           │
│    - 程序员可能不知道发生了昂贵的复制   │
├─────────────────────────────────────────┤
│ 2. 所有权不明确                          │
│    - s1 和 s2 都指向相同的数据？          │
│    - 谁负责释放？两个都释放？双重释放！  │
│    - 只有一个释放？另一个怎么办？        │
└─────────────────────────────────────────┘

Rust 的选择：
• 默认移动（零成本）
• 需要复制时显式调用 clone()
• 明确、可控、高效
```


## 显式深拷贝：clone()

```rust
fn main() {
    let s1 = String::from("hello");

    // 显式深拷贝
    let s2 = s1.clone();

    // 现在 s1 和 s2 都有自己独立的堆数据
    println!("s1 = {}, s2 = {}", s1, s2);  // 都有效

    // 修改 s2 不影响 s1
    let mut s2 = s2;
    s2.push_str(" world");
    println!("s1 = {}", s1);  // hello
    println!("s2 = {}", s2);  // hello world
}
```


## clone() 的内存布局

```
clone() 之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            └─────────┘
│ s2   │            ┌─────────┐
│ ptr  │ ─────────→ │ "hello" │
│ len  │            │ (副本)  │
│ cap  │            │         │
└──────┘            └─────────┘

关键点：
• s1 和 s2 指向不同的堆数据
• 数据被完整复制了一份
• 两者互不影响
• 各自在离开作用域时释放自己的数据
```
