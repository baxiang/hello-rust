# 移动语义

> 理解所有权的转移

---

## 移动 vs 复制

```rust
fn main() {
    // 对于 String 类型：移动语义
    let s1 = String::from("hello");
    let s2 = s1;  // 移动所有权

    // println!("{}", s1);  // ❌ 错误：s1 已无效
    println!("{}", s2);  // ✅ 正确

    // 对于整数类型：复制语义（Copy trait）
    let x = 5;
    let y = x;  // 复制值

    println!("x = {}, y = {}", x, y);  // 都有效
}
```

---

## 移动语义图解

```
移动前：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
└──────┘            └─────────┘

let s2 = s1;  之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ (无效)     │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            │         │
│ s2   │ ───────────┘         │
│ ptr  │                      │
│ len  │                      │
│ cap  │                      │
└──────┘                      │

关键点：
• s2 获得了指向堆数据的指针
• s1 被标记为无效（不能使用）
• 数据没有被复制！
• 当 s2 离开作用域时，数据被释放
```

## 为什么 Rust 不自动深拷贝？

```
假设 Rust 自动深拷贝：
let s1 = String::from("hello");
let s2 = s1;  // 如果自动复制堆数据...

问题：
┌─────────────────────────────────────────┐
│ 1. 性能问题                              │
│    - 堆数据可能很大（几 MB 甚至几 GB）     │
│    - 每次赋值都要复制，非常慢           │
│    - 程序员可能不知道发生了昂贵的复制   │
├─────────────────────────────────────────┤
│ 2. 所有权不明确                          │
│    - s1 和 s2 都指向相同的数据？          │
│    - 谁负责释放？两个都释放？双重释放！  │
│    - 只有一个释放？另一个怎么办？        │
└─────────────────────────────────────────┘

Rust 的选择：
• 默认移动（零成本）
• 需要复制时显式调用 clone()
• 明确、可控、高效
```

## 显式深拷贝：clone()

```rust
fn main() {
    let s1 = String::from("hello");

    // 显式深拷贝
    let s2 = s1.clone();

    // 现在 s1 和 s2 都有自己独立的堆数据
    println!("s1 = {}, s2 = {}", s1, s2);  // 都有效

    // 修改 s2 不影响 s1
    let mut s2 = s2;
    s2.push_str(" world");
    println!("s1 = {}", s1);  // hello
    println!("s2 = {}", s2);  // hello world
}
```

---

## clone() 的内存布局

```
clone() 之后：
栈                     堆
┌──────┐            ┌─────────┐
│ s1   │ ─────────→ │ "hello" │
│ ptr  │            │         │
│ len  │            │         │
│ cap  │            │         │
├──────┤            └─────────┘
│ s2   │            ┌─────────┐
│ ptr  │ ─────────→ │ "hello" │
│ len  │            │ (副本)  │
│ cap  │            │         │
└──────┘            └─────────┘

关键点：
• s1 和 s2 指向不同的堆数据
• 数据被完整复制了一份
• 两者互不影响
• 各自在离开作用域时释放自己的数据
```
