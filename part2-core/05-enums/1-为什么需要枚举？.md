## 11.1 为什么需要枚举？

### 问题场景

```rust
// 问题：表示方向
// 方案 1：使用整数（不安全）
let direction: i32 = 0;  // 0 代表什么？1、2、3 呢？

// 方案 2：使用字符串（不高效）
let direction: &str = "north";  // 可能拼写错误

// 方案 3：使用枚举（安全、高效）
enum Direction {
    North,
    South,
    East,
    West,
}
let direction = Direction::North;  // 类型安全，有补全
```

### 枚举的优势

```
┌─────────────────────────────────────────────────────┐
│              枚举的优势                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 类型安全                                         │
│     • 只能是定义的几个值之一                         │
│     • 编译器检查，防止无效值                         │
│                                                     │
│  2. 表达力强                                         │
│     • 每个变体可以携带不同数据                       │
│     • 可以精确描述状态                               │
│                                                     │
│  3. 穷尽检查                                         │
│     • match 必须处理所有情况                         │
│     • 添加新变体时，编译器提示更新代码               │
│                                                     │
│  4. 零成本抽象                                       │
│     • 和普通整数一样高效                             │
│     • 没有运行时开销                                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```
