## 为什么错误处理很重要？

### 现实世界的程序总会出错

```rust
// 可能失败的操作
std::fs::File::open("config.txt");    // 文件可能不存在
std::net::TcpStream::connect("localhost:8080");  // 服务器可能离线
"123".parse::<i32>();                  // 字符串可能不是有效数字
vec![1, 2, 3][10];                     // 索引可能越界
```

### 错误处理的目标

```
┌─────────────────────────────────────────────────────┐
│              良好的错误处理                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 程序健壮性                                       │
│     • 遇到错误时不会崩溃                            │
│     • 能够优雅地降级或恢复                          │
│                                                     │
│  2. 用户友好                                        │
│     • 提供清晰的错误信息                            │
│     • 指导用户如何解决问题                          │
│                                                     │
│  3. 易于调试                                        │
│     • 记录足够的上下文信息                          │
│     • 追踪错误来源                                  │
│                                                     │
│  4. 类型安全                                        │
│     • 编译器强制处理错误                           │
│     • 不会遗漏错误情况                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```
