## 15.9 常见错误

### 错误 1：缺少必要的约束

```rust
// ❌ 错误：T 没有实现 Display
// fn print_item<T>(item: T) {
//     println!("{}", item);  // 编译错误！
// }

// ✅ 正确
fn print_item<T: std::fmt::Display>(item: T) {
    println!("{}", item);
}
```

### 错误 2：类型参数不一致

```rust
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    // ❌ 错误：x 和 y 必须是同一类型
    // let p = Point { x: 5, y: 4.0 };

    // ✅ 正确：使用两个泛型参数
    let p = Point { x: 5, y: 4.0 };
}

// 修正版本
struct Point<T, U> {
    x: T,
    y: U,
}
```

### 错误 3：在 struct 定义时添加约束

```rust
// ❌ 错误：不要在结构体定义中添加不必要的约束
// struct Point<T: Copy> {
//     x: T,
//     y: T,
// }

// ✅ 正确：约束应该在 impl 块中
struct Point<T> {
    x: T,
    y: T,
}

impl<T: Copy> Point<T> {
    fn get_x(&self) -> T {
        self.x  // Copy 允许直接复制
    }
}
```

### 错误 4：泛型与生命周期

```rust
// ❌ 错误：忽略生命周期
// fn get_first<T>(slice: &[T]) -> &T {
//     &slice[0]
// }

// ✅ 正确：显式声明生命周期
fn get_first<'a, T>(slice: &'a [T]) -> &'a T {
    &slice[0]
}

// 或者让编译器推断
fn get_first_auto<T>(slice: &[T]) -> &T {
    &slice[0]
}
```

### 错误 5：滥用泛型

```rust
// ❌ 不推荐：过度泛化，代码复杂
fn process<T, U, V, W>(t: T, u: U, v: V, w: W) -> String
where
    T: Clone + Display,
    U: Into<String>,
    V: From<T>,
    W: PartialEq<V>,
{
    String::new()
}

// ✅ 推荐：简单明确
fn process(input: &str) -> String {
    format!("Processed: {}", input)
}
```
