## 17.5 生命周期省略规则

### 什么是省略规则

```
Rust 编译器在某些情况下可以自动推断生命周期，
无需显式标注。这称为"生命周期省略"（Lifetime Elision）。

┌─────────────────────────────────────────────────────┐
│           三条省略规则                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  规则 1：每个引用参数获得独立的生命周期            │
│  fn foo(x: &str, y: &str)  →  fn foo<'a, 'b>(x: &'a str, y: &'b str)
│                                                     │
│  规则 2：只有一个输入生命周期时，赋值给所有输出    │
│  fn foo(x: &str) -> &str  →  fn foo<'a>(x: &'a str) -> &'a str
│                                                     │
│  规则 3：方法中&self 的生命周期赋值给输出          │
│  fn foo(&self) -> &str  →  fn foo<'a>(&'a self) -> &'a str
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 省略示例

```rust
// ✅ 无需标注：单参数，自动应用规则 2
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}
// 等价于：
// fn first_word<'a>(s: &'a str) -> &'a str

// ✅ 无需标注：方法，自动应用规则 3
impl<'a> ImportantExcerpt<'a> {
    fn get_part(&self) -> &str {
        self.part
    }
}
// 等价于：
// fn get_part<'a>(&'a self) -> &'a str

// ❌ 需要标注：多个参数，无法确定关系
// fn longest(x: &str, y: &str) -> &str  // 错误！
```

### 省略规则应用

```rust
// 示例 1：单参数函数
fn print_and_return(s: &str) -> &str {
    println!("{}", s);
    s  // 返回输入，生命周期相同
}

// 示例 2：多个参数但无返回引用
fn print_both(x: &str, y: &str) {
    println!("{} {}", x, y);
}

// 示例 3：方法
struct Data {
    value: String,
}

impl Data {
    // &self 生命周期自动赋值给返回
    fn get(&self) -> &str {
        &self.value
    }

    // &mut self 同理
    fn get_mut(&mut self) -> &mut str {
        &mut self.value
    }
}
```
