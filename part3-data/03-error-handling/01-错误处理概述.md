## 14.2 Rust 的错误分类

```
┌─────────────────────────────────────────────────────┐
│              Rust 错误分类                           │
├─────────────────────────────────────────────────────┤
│                                                     │
│  可恢复错误 (Recoverable Error)                     │
│  ├── 预期可能发生的错误                             │
│  ├── 可以优雅处理的错误                            │
│  ├── 使用 Result<T, E> 处理                         │
│  └── 例：文件不存在、网络超时、解析失败            │
│                                                     │
│  不可恢复错误 (Unrecoverable Error)                 │
│  ├── 严重的程序错误                                │
│  ├── 无法继续执行的状态                            │
│  ├── 使用 panic! 终止程序                          │
│  └── 例：数组越界、除零、断言失败                  │
│                                                     │
│  Rust 哲学：                                        │
│  • 默认使用 Result 处理可恢复错误                  │
│  • panic! 仅用于真正不可恢复的情况                │
│  • 让错误尽早暴露（编译时而非运行时）              │
│                                                     │
└─────────────────────────────────────────────────────┘
```
