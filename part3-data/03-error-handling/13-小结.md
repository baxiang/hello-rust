## 14.13 小结

### 错误处理方法对比

| 方法 | 行为 | 适用场景 |
|------|------|----------|
| `panic!` | 终止程序 | 不可恢复错误 |
| `unwrap()` | 成功返回值，失败 panic | 测试、原型 |
| `expect(msg)` | 成功返回值，失败带消息 panic | 有明确预期时 |
| `match` | 显式处理所有情况 | 需要精细控制 |
| `if let` | 只处理成功/失败一种情况 | 另一种情况简单时 |
| `?` | 传播错误 | 函数返回 Result |

### 核心要点

1. **区分可恢复和不可恢复错误**
2. **使用 Result 处理预期错误**
3. **提供有意义的错误信息**
4. **不要忽略错误**
5. **使用自定义错误类型提高可维护性**
