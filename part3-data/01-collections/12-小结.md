## 12.12 小结

### 数组与 Vec 对比

| 特性 | 数组 | Vec |
|------|------|-----|
| 类型 | `[T; N]` | `Vec<T>` |
| 长度 | 固定（编译时） | 可变（运行时） |
| 存储 | 栈上（通常） | 堆上 |
| 大小 | `N * sizeof(T)` | `len * sizeof(T) + 开销` |
| 性能 | 稍快 | 稍慢（但更灵活） |
| 方法 | 较少 | 丰富 |

### 常用方法速查

| 方法 | 数组 | Vec | 说明 |
|------|------|-----|------|
| `len()` | ✅ | ✅ | 获取长度 |
| `is_empty()` | ✅ | ✅ | 判断为空 |
| `first()/last()` | ✅ | ✅ | 首尾元素 |
| `get(i)` | ✅ | ✅ | 安全访问 |
| `iter()/iter_mut()` | ✅ | ✅ | 迭代器 |
| `push()/pop()` | ❌ | ✅ | 末尾操作 |
| `insert()/remove()` | ❌ | ✅ | 插入/删除 |
| `extend()` | ❌ | ✅ | 批量添加 |
| `clear()` | ❌ | ✅ | 清空 |

### 关键要点

1. **数组**用于固定大小的集合，存储在栈上，性能最优
2. **Vec**用于动态集合，可自动增长，使用最广泛
3. 索引访问会进行边界检查，越界会 panic
4. 使用 `get()` 方法可以安全访问，返回 `Option`
5. Vec 扩容时会重新分配内存，预分配容量可优化性能
6. 函数参数优先使用切片 `&[T]`，可同时接受数组和 Vec
